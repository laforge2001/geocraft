Horizon Cascade (d->t->d)

Horizon Cascade converts horizons from depth to time using one velocity, then converts them back to depth using a second velocity.

Overview of the Horizon Cascade Algorithm

The Horizon Cascade (d->t->d) algorithm uses velocity volumes to perform depth-to-time followed by time-to-depth conversions on horizons. You may convert several horizons at once, and can use an Area of Interest in the calculation to limit the processing area.

Cascading a Horizon from Depth to Time followed by Time to Depth

  1. Load ModSpec grid and volume data into GeoCraft.
  2. In the Depth Horizons field, click Add.
  3. In the Selection Dialog, highlight the depth horizons you want to use in the calculation and click OK.
  4. In the Depth Horizons field, select individual horizons and click Up and Down to sort the horizons in the calculation order. GeoCraft gives preferential treatment to the top-most horizon in the list during the calculations. If you selected a horizon you do not want, highlight the horizon and click Remove.
  5. Select an Interval Velocity Volume (V1) , the volume that will convert horizons from depth to time.
  6. Select an Interval Velocity Volume (V2), the volume that will convert horizons from time to depth.
  7. Optional; only perform this step if you want to use an Area of Interest in the calculation. Select the Use Area of Interest? check box.
  8. Optional Select an Area of Interest.
  9. Select a Conversion Method:
    • Cell-based - vertical velocities are held constant between input values
    • Knee-based - vertical velocities are linearly interpolated between input values
  10. Accept the default in the Output Depth to Time Suffix field, or type in a new value.
  11. Accept the default in the Output Time to Depth Suffix field, or type in a new value.
  12. Click Run.

A progress bar is displayed while the conversions run. When the job completes, output horizons are in the Repository window.

You may now plot and view the data in one of the GeoCraft viewers.
