Velocity Flood

Velocity Flood modifies velocities in regions above, below, or between horizons. Flood velocities may be defined as constant, gradient, or another dataset.

Velocity flooding can fail when bounding horizons have holes, different extents, or when they cross over. These problems can be very difficult to spot in the output volume. Use the Velocity Model Checker algorithm to identify problems before doing the flood. The Velocity Model Checker algorithm is in the Horizon folder in the algorithm tree . This algorithm creates a horizon that which is color coded to locate areas of potential problems in the Map Viewer.

Because velocity flooding can take a long time to complete, some flooding algorithms allow users to preview results in the 3D Viewer for a single inline and xline prior to flooding the entire input volume. These algorithms are located in the Algorithms tree under the Preview Algorithms folder.
