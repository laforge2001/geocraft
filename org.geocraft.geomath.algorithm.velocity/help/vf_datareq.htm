Velocity Flooding Data Requirements

Caution: If your data does not meet one of the following requirements, GeoCraft displays a message in the Error Log, and the program does not run.

Data used for velocity flooding must meet the following requirements:

  • If you are flooding between two horizons, both horizons must have the same grid sampling.
  • Velocity units for flood data (either constant, gradient, or dataset) must be consistent. For example, if the input volume is in feet per second, the flood dataset must also be in feet per second.
  • Vertical units for the input velocity volume and horizons must match. For example, if the input velocity is a depth-based volume, all horizons must also be depth-based.
  • Velocity gradient flooding is only allowed on depth input volumes.
  • Velocity gradient units must be either feet/sec/foot or meter/sec/meter.
  • The output volume name is not required to be a unique filename and you can overwrite existing data.
  • All input parameters must be unit consistent with the input velocity volume.
