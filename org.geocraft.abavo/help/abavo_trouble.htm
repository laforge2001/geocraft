Troubleshooting ABAVO Problems

This topic provides descriptions for some common ABAVO problems, and tells you how to resolve each problem.

Symptom Cause Resolution
ABAVO is taking a long time to draw crossplots. You may be creating one or more crossplots that contain data sets with more than 200,000 points. See Reducing the Amount of Time ABAVO Takes to Draw a Crossplot in ABAVO: Tips for Using the Plug-in and Improving Its Performance
The Color Class Background does not display when ABAVO draws a background and event ellipse. You must use the same Active Series data number and the same regression line to create the background and event ellipses. If you select a different regression line or Active Series of data, the background color will not display. See Use the Same Active Series Data and Regression Line to Create the Background and Event Ellipses in ABAVO: Tips for Using the Plug-in and Improving Its Performance
The Color Class Background and Region Bounds do not display when ABAVO creates the background and event ellipses. When the Class Background or Region Bounds do not display, it is usually because some of the region lines are crossed.

Perform the following procedure to resolve the issue.

  1. Turn off the data points for the background and event ellipse data.
  2. See if any region lines cross each other.
  3. If region lines do cross each other, correct the problem:
    1. Perform the steps in Changing Line Position in a Crossplot.
    2. Repeat the procedure in step a until all region lines are no longer crossed.
    3. Press mouse button 3 to refresh the ABAVO crossplot window.
The Color Class Background does not match the Region Lines. The region lines within the plot need to be updated. Position the cursor in the Crossplot window and click mouse button 3 (MB3) to refresh the plot.
I updated the region lines within the Crossplot, but the Color Class Background still does not match the region lines. A setting on the Ellipse Model table needs to be changed.
  1. Click Edit Ellipse Model.
  2. Clear the Symmetry Lock on Region Bounds check box.
  3. Click Apply.
I lost my existing data sets (or, existing data sets were overwritten when a crossplot was generated). You ran an ABAVO calculation before you selected a new Active Series number in the Crossplot window. The previous data that was associated with the Active Series number is lost; it cannot be retrieved. Select a new Active Series number before you run each new ABAVO job.
The Java Virtual Machine (JVM) runs out of memory and I cannot create crossplots. You may be creating one or more crossplots that contain data sets with more than 200,000 points. See Reducing the Amount of Time ABAVO Takes to Draw a Crossplot in ABAVO: Tips for Using the Plug-in and Improving Its Performance
