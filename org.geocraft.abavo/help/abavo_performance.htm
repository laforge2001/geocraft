ABAVO: Tips for Using the Plug-in and Improving Its Performance

The following tips will help you use ABAVO and ensure that the plug-in runs at its peak performance:

Tip Explanation and Further Details
Reduce the Amount of Time ABAVO Takes to Draw a Crossplot

Make sure that each crossplot contains less than 200,000 points, particularly if you are creating several crossplot data sets at once. If several of your data sets exceed 200,000 points, ABAVO draws the crossplots slowly, and the Java Virtual Machine (JVM) may run out of memory.

Estimate the number of points in a crossplot with this equation:

number of inlines x number of crosslines x number of Time samples = number of points in crossplot

If the total number of points is greater than 200,000, use one of the following options to reduce the points in the crossplot:

  • use the Peaks & Troughs data mode
  • reduce the time window for your calculation
  • use an Area of Interest (AOI) to limit the inline and crossline range
  • delete any data sets that you do not need from the crossplot
Use the Same Active Series Data and Regression Line to Create the Background and Event Ellipses If you want to construct a background and an event ellipse, you must use the same Active Series data number and regression line to create both ellipses. If you use a different Active Series number or regression line in either ellipse, the color class background will not display.
View Plot Statistics and Processing Information While ABAVO Runs Calculations ABAVO automatically computes plot processing information and regression statistics as each data set is completed. You can view the data results in the GeoCraft Error Log.
