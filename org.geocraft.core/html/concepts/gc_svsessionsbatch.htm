Saving a GeoCraft Session for Batch

Use this procedure to save a GeoCraft session in order to run it later in batch mode, so that you can create and run batch processes in GeoCraft.

Note: The Save Session as Batch command saves any project-specific preferences that you set during the session. When GeoCraft is restarted later in a batch run, those preferences are restored before the repository is built, so that unit system preferences in the original workspace are maintained.

To Save a GeoCraft Session as a Batch File

  1. Launch GeoCraft.
  2. Optional Load data into GeoCraft.
  3. Make any changes to the current GeoCraft environment (for example: change the default layout for GeoCraft; or change the preferences).
  4. Open an algorithm.
  5. Populate all required fields within the algorithm panel and enter optional data that you want to run within the algorithm.
  6. Select File => Save Session as Batch.
  7. In the Batch Saveset dialog box, type a Name for the batch session.
  8. In the Save in folder accept the default value, or browse to another folder.
  9. Click OK.

GeoCraft saves the session batch file with a *.gcb filename extension. Note that if more than one algorithm are visible (opened) in the center parameter panel of the GeoCraft window, all algorithms are saved in the batch session file.

Caution: The GeoCraft team recommends using only one algorithm in a batch process. If you use more than one algorithm, the algorithms will be executed sequentially but unfortunately results may not be what you expect, particularly if the output from one algorithm must feed into another. Currently this type of batch operation cannot be done. This restriction will be lifted when a workflow editor is provided.
