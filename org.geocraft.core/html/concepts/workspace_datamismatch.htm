GeoCraft Sessions: Loading Data that Does Not Match Unit Preferences

Project data that you load into GeoCraft should match the unit preferences defined in your GeoCraft workspace. If you load data into GeoCraft that is in different units, GeoCraft will load the data but will convert the units to match the session unit system. GeoCraft does not change projection coordinate systems, nor data units, such as velocity in m/sec to ft/sec.

See the following topics for a detailed explanation about how GeoCraft handles units, and unit mismatches when data are loaded:

  • Loading Data that Does Not Match Unit Preferences
  • Unit Preference Settings and How GeoCraft Performs Data Unit Conversions

When data is about to be loaded from a different unit system, GeoCraft will display a warning message in the Load dialog box, as shown below.

Generally, units should not matter. However, if a session could be saved in one unit system and then restored later in another unit system, there is the potential for X,Ys to be in error. To help prevent this GeoCraft sessions have a fixed unit system which cannot be changed. If another session is restored, the unit system of that session is maintained.
