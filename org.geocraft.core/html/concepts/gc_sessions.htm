GeoCraft Sessions

When you launch GeoCraft using a particular workspace, that GeoCraft instance is referred to as a GeoCraft session. You can save the session's state at any time and restore it later. A session can contain any of the following items. Session files retain the state of your current GeoCraft environment, including:

  • Repository contents
  • data that was displayed in the GeoCraft Viewers, and the settings at which that data was displayed
  • GeoCraft algorithms that were open and in use
  • user preferences, including the unit system
  • the following views:
    • ABAVO
    • 3D View
    • Map View
    • Section View
    • Wavelet View
    • Well Log View

Currently, session files do not restore Histogram and Scatter Plot views. Caution: If you share your session with others, they must have file system permissions to access files in that data store.

Quitting a GeoCraft Session

Caution If you click the X in the top right corner of the GeoCraft window to quit the session, this closes all separate plot windows immediately, and any plots that you were viewing in a separate window cannot be saved as part of your session state when you exit GeoCraft. Always use File => Exit on the main menu to quit GeoCraft.

When you select File => Exit to quit GeoCraft, you are prompted to save the last active session in which you worked. If you are working in an existing session, the filename and path are populated with the existing session information.

Handling Units of Measure within a Session

When you launch GeoCraft, you will be prompted to specify the path of a workspace. If you specify a workspace that does not exist, it will be created, and you must also specify the unit system preference for your session. When you load new data, GeoCraft will convert X,Y and Depth (only) units to the unit system of the session.

Advantages to Using GeoCraft Sessions

GeoCraft sessions offer several advantages; because of the GeoCraft session functionality, you can easily:

  • pick up where you left off in your work. GeoCraft saves your session in its current state; therefore, if you need to leave a project or a geophysical problem on which you are working for awhile, GeoCraft saves all of your data and the related tools (algorithms, viewers, calculators, etc.) you were using to resolve your problem in their current state. When you are ready to return to work, everything is waiting for you, just as you left it.
  • share your work with others. The GeoCraft session file is a portable XML file; you can let other users point to your session file and they will be able to work in the exact session environment that you set up.
  • return to a starting point in your work. For example, if you save your session file at a certain point and then perform several calculations or lay out several plots and realize that the more recent work you have done is not what you wanted to do, you can always restore your prior saved session, and return to a "clean slate" within your work.
