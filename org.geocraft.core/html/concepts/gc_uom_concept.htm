Unit Preference Settings and How GeoCraft Performs Data Unit Conversions

When you start GeoCraft, you must define the default unit of measure preferences for X,Y, and depth. See Setting Unit Preferences for instructions.

Note: GeoCraft does not convert X,Ys or Latitude and Longitudes to different projection coordinate systems. All data must be in the same geographic coordinate system.

Warning: GeoCraft always converts X,Y and depth units to the unit preference of the current workspace. GeoCraft performs this conversion for all wells, horizons, and the X,Y coordinates of volumes. However GeoCraft never converts data units, such as velocities. Input velocities in m/sec are not converted to ft/sec.

GeoCraft Warning When X,Y Units Do Not Match

GeoCraft displays this warning message when you are about to import data that does not match your session units.

How GeoCraft Converts Units

If you import data measured in units that do not match the unit system that you set up for the session in which you are working, GeoCraft displays a warning message. When you click OK to import the data, GeoCraft automatically converts the data to the units of measure that you have set as your default.

Data Type Conversion Details
Horizon GeoCraft converts X, Y coordinates and depth or time index values
X, Y coordinates of a volume GeoCraft converts the X, Y, coordinates of the volume from one data format to another so that they match your default preference (for example, data may be converted from meters to feet)
Depth or Time of Volume GeoCraft converts the depth or time index units for volumes if the data you are about to import does not match your default settings. For example, if your unit preference is set to feet, and you are going to import a volume with its depth measured in meters, GeoCraft converts the depth measurement to feet before the volume is imported.
