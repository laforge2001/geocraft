GeoCraft Session Files

GeoCraft creates two types of session files:

  • *.gcs: GeoCraft session files, which are stored in the GeoCraft workspace/sessions subdirectory
  • *.gcb: GeoCraft session batch files, which are stored in the GeoCraft workspace/gcbatch subdirectory

By default, GeoCraft session files are stored in subdirectories of your current workspace. However, you may store them in any directory you choose when saving a session.

GeoCraft session files are written in XML code, so it is possible for you to invalidate a session file if you make changes in the XML file (for example, by changing the names of keys within the file).

Caution! Do not edit the session files; you may lose or corrupt data within the session file and be unable to use it.

Managing GeoCraft Session Files

GeoCraft does not have an automated mechanism to delete, back up, or restore the GeoCraft session files. You must manage the GeoCraft session file directories and their contents. If you no longer need a session file, navigate to the directory in which the session file is stored, and manually delete that file.

Using Session Files

You can create as many session files as you want to in your working environment. A session is an instance of GeoCraft, and the state in which that instance was when you saved it.

Session files retain the state of your current GeoCraft environment, including:

  • Repository contents
  • data that was displayed in the GeoCraft Viewers, and the settings at which that data was displayed
  • GeoCraft algorithms that were open and in use
  • user preferences, including the session unit system preference
  • the following views, and any plots within the view:
    • ABAVO
    • 3D View
    • Map View
    • Section View
    • Wavelet View
    • Well Log Plot (for LAS wells)

Currently, session files do not restore the following views:

  • Histogram Plot
  • Scattergram Plot
