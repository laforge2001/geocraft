Before You Use GeoCraft

This topic contains brief descriptions of important GeoCraft concepts, features, and functionality that you should understand before you use GeoCraft.

Concept, Feature or Function Explanation and Further Details
Access geophysical data. GeoCraft can directly access geophysical data such as 2D or 3D seismic and well data
Specify a Workspace location for GeoCraft sessions and project data. See Specifying a Location for GeoCraft Sessions and Project Data for more details.
Set your GeoCraft user preferences. See Setting GeoCraft Preferences for more details.
Load geophysical data into GeoCraft; you must do this before you can use algorithms to perform calculations, or view data in any of the viewers. See Working with Data in GeoCraft for details.
Familiarize yourself with the GeoCraft perspectives, and how to switch between perspectives. See GeoCraft Perspectives for more details.
Familiarize yourself with the GeoCraft user interface elements. See GeoCraft User Interface Elements for more details.
Make sure you understand the fundamentals of the GeoCraft datastore repository. See Fundamentals of the GeoCraft Data Store Architecture for more details.
Make sure you follow GeoCraft file-naming conventions. All outputs for GeoCraft (for example: variables, volumes, and horizons) must have a unique file name. See GeoCraft File-naming Requirements for more details.
All file names in the GeoCraft Repository must be unique.

Existing data in a datastore or in the Repository may be overwritten. If a file already exists, GeoCraft displays a warning icon (a small yellow triangle). GeoCraft also displays a message that states the file currently exists. If you do not want to overwrite the data, you must change the output name before you click Run.
Make sure you understand how the GeoCraft Run in Background command works. Use this command when you want to send a job to the background and continue working. However, do not make changes to the data sets you are using for the calculation that you just sent to the background. See Running GeoCraft Processes in the Background for more details.
