Setting Unit of Measurement Preferences in GeoCraft

The first time you start GeoCraft, you will be prompted to specify a the unit of measure system (metric or imperial) for project data that is loaded into the GeoCraft session. Note: Unit of measure preferences are fixed when the session is created and cannot be changed. This feature prevents data in a single session from being loaded in imperial and metric coordinates. If you work with data in both unit systems, create separate sessions for metric and imperial data sets.

How GeoCraft Handles Conversions when Data or the Unit System does not Match the Session Unit System Preference

At times, you will load data into GeoCraft that does not match the unit of measure preferences that are set for that GeoCraft session. GeoCraft will display a warning message to let you know that the data you are loading does not match the unit system preferences that have been set for the GeoCraft session.

Then, depending upon the data type, GeoCraft either converts the data to the GeoCraft session's preferred unit of measure, or leaves the data in its existing unit of measure format.

GeoCraft Unit of Measure Conversion Rules

GeoCraft will always convert the following types of units to the unit preferences of the GeoCraft session:

  1. X,Y coordinates for all horizons (grids)
  2. depth units
  3. time units
  4. all well-related units
  5. X,Y coordinates of volumes

GeoCraft never converts data units, such as velocities. For example, an input velocities measured in in m/sec would not be converted to ft/sec.
