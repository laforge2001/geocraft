Naming and Equation Conventions for Multiples Algorithms

The Group Multiples algorithms create multiples as new horizons with a special naming convention in order to clearly specify the travel path for each multiple.

The form is always something like hm_NNN_MM, where the "hm" is the Horizon Prefix set by the user. The Ns represent the primary reflections at horizon number "N". The Ms represent reflections at horizon number "M" for pegleg paths. There will always be one less number of M's than N's. When reading a travel path the M's are always inserted between the N's. An M of 0 represents a surface reflection.

A few examples will clarify this scheme. Let the first input horizon be waterbottom, the second be top salt and the third be base salt.

  • hm_11_0 indicates travel time from waterbottom (i.e. N=1) up to the surface (i.e. M=0) and back (second N=1), or the first waterbottom multiple
  • hm_111_00 indicates travel time from waterbottom (N=1) up to the surface (M=0) and back, and then up to the surface and back a second time, or the second waterbottom multiple
  • hm_1111_000 indicates travel time from waterbottom up to the surface and back a total of 3 times, or the third waterbottom multiple
  • hm_22_0 indicates travel time from top salt (N=2) up to the surface (M=0) and back, or the first top salt multiple
  • hm_22_1 indicates travel time from top salt (N=2) up to the waterbottom (M=1) and back, or the first top salt, waterbottom multiple
  • hm_33_1 indicates travel time from base salt (N=3) up to the waterbottom (M=1) and back, or the first base salt, waterbottom multiple
  • hm_33_2 indicates travel time from base salt (N=3) up to the top salt horizon (M=2) and back, or the first top salt, base salt multiple
  • hm_3333_102 indicates travel time from base salt up to waterbottom (M=1), then back to base salt, then up to the surface (M=0) and back to base salt, and finally up to top salt (M=2) and back to base salt (N=3)
  • hm_321_00 indicates travel time from base salt (N=3) up to the surface (M=0), then down to top salt (N=2), then back up to the surface (M=0) and finally down to top salt (N=1)

More Examples of the Multiples Naming Convention

The actual equation used for each multiple can also be determined from the naming convention. For example suppose a multiple is named hm_331_10.

Naming Convention Interpretation
hm

this Horizon Prefix parameter set by the user
_ first underscore in the file name; every value between the first and second underscore is added in the multiples calculation
331 values representing the horizons that will be added in the multiples calculation
_ second underscore in the file name; everything value after this is subtracted
10 values representing the horizons that will be subtracted from the multiples calculation; remember that zero values are ignored by GeoCraft

Travel time calculation

Using variable 333_10 from this example, GeoCraft would perform the following calculation:

H3 + (H3 - H1) + H3
