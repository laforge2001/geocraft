Velocity Model Checker

Velocity Model Checker allows users to QC pairs of top and base salt horizons before using them in velocity flood. The algorithm will flag holes and crossovers by color coding the output horizon to give a map view of potential problem areas. Horizons that cross can be fixed using the Horizon Merge utility module.

Velocity flooding using top and base salt horizons is often problematic because these horizons are difficult to pick. They can have missing areas and sometimes they will cross. Additionally, there may be holes in one horizon but not the other. These are all problems for the Velocity Flood module.

Note: Output color assignments match the default Spectrum colormap used in the GeoCraft Map and 3D Viewers.

Overview of the Velocity Model Checker Algorithm

The Velocity Model Checker algorithm saves time by quickly checking for holes in horizon pairs without requiring you to complete an actual velocity flood model. You can also use the algorithm to highlight areas where the base horizon rises above the top horizon.

Using the Velocity Model Checker Algorithm

  1. Load two horizons into GeoCraft.
  2. Select a Top Horizon.
  3. Select a Base Horizon.
  4. Select a different color for each of the Parameters, so that you can analyze the differences when you view the model:
    • Holes in Base Horizon
    • Holes in Top Horizon
    • Holes in both Horizons
    • Base is Above Top (horizon)
    • No conflicts
  5. Accept the Name, or type in a new one. Make sure that the name complies with GeoCraft file-naming requirements.
  6. Click Run.

    7. GeoCraft runs the algorithm and stores the model horizon in the Repository, under the Grids folder.

  7. Use the GeoCraft Map Plot to view the results.
