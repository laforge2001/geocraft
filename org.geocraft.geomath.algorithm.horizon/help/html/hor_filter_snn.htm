Symmetric Nearest Neighbor Algorithm

The Symmetric Nearest Neighbor (SNN) filter algorithm compares each pixel inside a kernel with the pixel that is directly opposite of it inside the kernel. Linear filters work by convolution within a moving window that is called a kernel. The input pixel is at the center of the kernel. The nonzero part of the kernel is the kernel support.

Overview of the Symmetric Nearest Neighbor Algorithm

The SNN filter preserves the edges of a data set, and is very effective at noise reduction. SNN compares opposite pairs of pixels in the support, and selects the pixel from each pair that is closest in value to the central input pixel. Then, SNN returns the mean of the selected pixels.

Tip: The SNN filter is a good choice when you need to filter a faulted horizon, and you must preserve the edges of the horizon. SNN is a versatile filter.

Using the GeoCraft SNN Algorithm to Filter a Seismic Horizon

  1. Load seismic horizon data into GeoCraft.
  2. Select a Horizon.
  3. Accept the Size, or type in a new value. The size is the number of inlines and xlines in the sample.

    4. Tip: Size must be an odd number that is 3 or greater.

  4. Accept the Name, or type in a new one. Make sure that the name complies with GeoCraft file-naming requirements.
  5. Optional Type in Comments. Text typed in this field displays in the Propertyview when the new horizon is selected in the Repository.
  6. Click Run.

    7. GeoCraft runs the calculation, and creates the new, smoothed, horizon. The new horizon is in the Repository, under the Grids folder.

  7. Use the GeoCraft Map Plot to view the results.
