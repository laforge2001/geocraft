Group Multiples

Group Multiples computes specific, predefined multiples, including interbed multiples, from a list of input horizons ordered by increasing time or depth. Input horizons may be in either time or depth, but if they are in depth, an input velocity volume is also required.

Note: Predefined Set #1 allows up to 5 input depth or time horizons. Predefined Set #2 allows up to 3 input time horizons. GeoCraft creates multiples with a common naming convention and format to clearly specify peglegs (see predefined multiples for Set #1). The naming convention for output multiple horizons does not reflect the names of the input horizons. However, users can change the default prefix for output horizon names to do this.

Caution: Input horizons must be in order by increasing depth or time. If they are not, then resulting multiples may extend above zero time. If the input are depth horizons, the output will be clipped at zero depth by the time to depth conversion. Time horizons are not clipped at zero time.

WARNING: Group Multiples always writes the output horizons back to the input horizon datastore rather than storing them only in memory. This is necessary because many horizons can be created, and if these are large horizons, they could consume all available memory.

Overview of the Group Multiples Algorithm

Group Multiples can generate two different, predefined sets of multiples. If the input horizons are in time, results are in time and a velocity volume is not required. If the input horizons are in depth and you select Set #1, a velocity volume is required.

  • Set #1 converts up to 5 depth horizons to time, computes a specific set of time multiples, and then converts these horizons back to depth. Input depth horizons require an input velocity volume. If the input horizons are in time, results will be in time and a velocity volume is not required.
  • Set #2 computes all multiples for up to 3 input time horizons.

All horizons created by Group Multiples share a common naming convention to describe pegleg paths: common naming convention and equation format.

Generating Multiples from Selected Input Horizons

  1. Load horizon data into GeoCraft.
  2. Load a velocity volume (with units of velocity) into GeoCraft if the horizons are in depth.
  3. Click Add.

    4. A Selection Dialog pops up, listing all of the horizons loaded into GeoCraft.

  4. Select the horizons to use in the calculation and click OK.
  5. Sort horizons by increasing time or depth in the Horizons field, by selecting individual horizons and using the Up and Down buttons to move the horizons toward the top or bottom of the list.
  6. Select an interval Velocity Volume if horizons are in depth.
  7. Select a Predefined Group of Multiples:
    • Set #1 (Maximum of 5 Horizons) calculates predefined multiples for up to 5 depth or time horizons
    • Set #2 (Maximum of 3 Horizons) calculates all possible multiples for up to 3 time horizons
  8. Select a depth Conversion Method:
    • Cell-based - velocity is calculated analogous to a constant conversion method. Velocity is held constant vertically in depth or time between input velocities.
    • Knee-based - velocity is calculated analogous to a linear conversion method. Velocity is linearly interpolated vertically in depth or time between input velocities.
  9. Accept the Horizon Prefix, or type in a new value that complies with GeoCraft file-naming requirements.
  10. Click Run.

    11. GeoCraft calculates and saves the multiple horizons to the same location where the original horizons are located. GeoCraft creates multiples with a common naming convention and equation format. The new multiples populate into the Repository window, under the Grids folder.

  11. Use the GeoCraft Map Plot to view the results.
