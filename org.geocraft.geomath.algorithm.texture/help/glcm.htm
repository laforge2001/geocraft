Horizon Texture (GLCM) Algorithm

The Horizon Texture algorithm computes horizon texture attributes using the Gray Level Co-occurrence Matrix (GLCM) method. Co-occurrence is a measure of how often one color occurs next to a similar color or a different color after quantization into a fixed (64) number of gray levels. Smooth textures will have fewer transitions. Users control the search distance in grid cells and the size of the region to scan. For more information, see the tutorial on GLCM at www.fp.ucalgary.ca/mhallbey/the_glcm.htm. The following texture attributes are computed and output as grids:

  • entropy: indicates the amount of disorder (unexpected change) within the image
  • homogeneity: indicates the overall smoothness of the image
  • inertia: indicates the amount of local variation within the image
  • quantized: indicates the amount of compression within the image
  • energy: indicates the textural uniformity within the image

Using the Horizon Texture Algorithm to Compute Texture Attributes

  1. Load horizon data into GeoCraft.
  2. Select a Horizon.
  3. Set the Search distance . The number you type in this field is the radius, in grid cells, that is the offset used to compute the GLCM. In general the smaller the number, the smoother the results.
  4. Set the Volume distance. The number you type in this field is the size of the region, in grid cells, used to compute the GLCM. In general the larger the number, the smoother the results.
  5. Type an output Name.
  6. Optional Type text in the Comments field. Text typed in this field is displayed in the Properties region when an entity is selected in the Repository.
  7. Click Run.

    8. Each output grid is stored in the Repository under the selected input horizon.

  8. Use the GeoCraft Map Plot to view the new grids.
