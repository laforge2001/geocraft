Using a Scale Factor to Rescale Seismic Data for Conversion in the MVXP Algorithm

MVXP reduces seismic traces to an average median value of 1, so the algorithm could be used to rescale seismic data from 32-bit to 8- or 16-bit data, which would save disk space yet maintain true amplitudes.

You must set the Scale Factor larger than 1 to rescale the MVXP output. If the Scale Factor is less than 1, integer data will get clipped. Use the following Scale Factor values for data conversion:

  • To convert 32-bit float data into 8-bit data, enter 5.08 in the Scale Factor field. This clips the data at 25 times the median (5.08 is 127/25) so that the output ranges from -127 to 128.
  • To convert 32-bit float data into 16-bit data, enter 1310.68 in the Scale Factor field. This clips the data at 25 times the median (1310.68 is 32767/25).
